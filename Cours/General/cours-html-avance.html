La structure de toute page web — balises, attributs, et où aller plus loin.

Module 02 Fondations Balises avancées Attributs Ressources
1
Qu'est-ce que le HTML ?

HTML signifie HyperText Markup Language. Ce n'est pas un langage de programmation — c'est un langage de balisage. Il ne calcule rien. Il ne réagit à rien. Il décrit le contenu d'une page : ce qui est un titre, ce qui est un paragraphe, ce qui est une image, ce qui est un lien.

Le navigateur lit ton HTML et construit ce qu'on appelle le DOM (Document Object Model) — une représentation en arbre de tous les éléments. C'est sur ce DOM que le CSS et le JavaScript viennent travailler.

La relation entre les trois langages du web
🏗️ HTML — La structure : les murs, les pièces, les portes
🎨 CSS — Le style : la peinture, les meubles, la décoration
⚙️ JavaScript — Le comportement : l'électricité, la plomberie, les interrupteurs
💡 Un HTML bien structuré, sans CSS ni JavaScript, reste une page lisible et utilisable. C'est la fondation — si elle est solide, tout le reste s'appuie dessus correctement.
2
Anatomie d'une balise HTML

Tout en HTML repose sur les balises. Une balise est un mot-clé entouré de chevrons < >. La plupart vont par paire : une balise ouvrante et une balise fermante.

<!-- Structure complète d'un élément HTML --> <balise attribut="valeur">contenu visible</balise> <!-- Exemples concrets --> <p class="intro">Bonjour le monde</p> <a href="https://exemple.com" target="_blank">Cliquez ici</a> <img src="photo.jpg" alt="Description"> <!-- balise auto-fermante -->

Certaines balises n'ont pas de contenu et se ferment d'elles-mêmes : <img>, <br>, <hr>, <input>, <meta>, <link>. On les appelle balises vides ou void elements.

⚠️ Oublier de fermer une balise est l'une des erreurs les plus fréquentes. Le navigateur essaie de corriger seul — mais pas toujours comme tu l'espères. Prends l'habitude d'ouvrir ET fermer chaque balise immédiatement.
3
Structure complète d'une page HTML

Toute page HTML correcte commence par la même ossature. Chaque ligne a un rôle précis.

<!-- Déclaration de type de document --> <!DOCTYPE html> <html lang="fr"> <head> <!-- Infos pour le navigateur --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Titre de la page</title> <link rel="stylesheet" href="style.css"> </head> <body> <!-- Tout ce qui est visible --> <header>...</header> <main>...</main> <footer>...</footer> </body> </html>
<head> Invisible dans la page. Contient les métadonnées, le titre de l'onglet, les liens vers CSS et polices.
<body> Tout ce que l'utilisateur voit et interagit. Structure, textes, images, formulaires — tout est ici.
🏷️ Balises sémantiques
4
La sémantique — donner du sens au HTML

Avant HTML5, les développeurs construisaient leurs pages avec des <div> partout. HTML5 a introduit des balises sémantiques : des balises qui décrivent leur rôle, pas seulement leur apparence.

Un moteur de recherche, un lecteur d'écran pour malvoyants, ou un autre développeur comprendra immédiatement la structure de ta page si tu utilises les bonnes balises.

BaliseRôle
<header>
En-tête de page ou de section. Logo, titre principal, navigation.
<nav>
Bloc de navigation principale. Liens vers les autres pages.
<main>
Contenu principal unique de la page. Un seul <main> par page.
<article>
Contenu autonome et réutilisable : article de blog, carte produit, post.
<section>
Groupe thématique de contenu. Doit avoir un titre (<h2> à <h6>).
<aside>
Contenu secondaire : barre latérale, encadré, publicité, liens connexes.
<footer>
Pied de page ou de section. Mentions légales, liens utiles, contact.
<figure>
Contenu illustratif autonome (image, schéma, code). Associé à <figcaption> pour la légende.
<time>
Date ou heure lisible par les machines. Attribut datetime pour le format standard.
<address>
Informations de contact (email, adresse postale, téléphone).
<!-- Exemple de structure sémantique complète --> <body> <header> <nav><a href="/">Accueil</a></nav> </header> <main> <article> <h1>Mon article</h1> <time datetime="2025-04-18">18 avril 2025</time> <p>Contenu...</p> <figure> <img src="schema.png" alt="Schéma explicatif"> <figcaption>Figure 1 — Schéma</figcaption> </figure> </article> <aside>Articles connexes...</aside> </main> <footer> <address>contact@monsite.fr</address> </footer> </body>
🎯 La règle d'or : utilise <div> uniquement pour les conteneurs de style qui n'ont pas de signification sémantique. Pour tout le reste, il existe une balise qui dit ce que tu veux dire.
✍️ Balises de texte & mise en forme
5
Titres, texte et mise en valeur

HTML dispose d'une hiérarchie de titres de <h1> à <h6>, et plusieurs balises pour enrichir le sens du texte.

BaliseSens et usage
<h1> … <h6>
Titres hiérarchiques. Un seul <h1> par page — c'est le titre principal.
Ne pas sauter de niveau (h1→h3 sans h2)
<p>
Paragraphe. Crée automatiquement un espace avant et après.
<strong>
Importance forte. Affiché en gras, mais signifie aussi "important" pour les lecteurs d'écran.
Différent de <b> qui est purement visuel
<em>
Emphase/insistance. Affiché en italique, signifie "accent particulier" sémantiquement.
Différent de <i> qui est purement visuel
<mark>
Texte surligné/mis en évidence (fond jaune par défaut). Pertinent dans un contexte de recherche.
<del> / <ins>
Texte supprimé (barré) et inséré (souligné). Indique des modifications de contenu.
<abbr>
Abréviation. L'attribut title affiche l'explication au survol de la souris.
<blockquote>
Citation longue mise en retrait. L'attribut cite référence la source.
<cite>
Titre d'une œuvre citée : livre, film, chanson, article.
<code>
Code informatique en ligne. Police monospace. Pour les blocs de code, utilise <pre><code>.
<kbd>
Touche de clavier. Exemple : appuyez sur Ctrl+S
<sup> / <sub>
Exposant (10²) et indice (H₂O). Utile pour les formules scientifiques.
<small>
Texte secondaire, mentions légales, copyright. Taille réduite sémantiquement.
<p> La formule de l'eau est H<sub>2</sub>O. <abbr title="HyperText Markup Language">HTML</abbr> est né en <mark>1991</mark>. Prix : <del>29€</del> <ins>19€</ins> </p> <blockquote cite="https://source.com"> "La simplicité est la sophistication suprême." </blockquote>
6
Listes : ul, ol, dl

HTML propose trois types de listes, chacune avec un usage précis :

TypeBaliseUsage
Non ordonnée <ul> + <li>
Liste à puces. L'ordre n'a pas d'importance.
Ordonnée <ol> + <li>
Liste numérotée. Étapes, classements, instructions séquentielles.
Définition <dl> + <dt> + <dd>
Paires terme/définition. Glossaires, FAQ, métadonnées.
<!-- <ol> avec attributs start et type --> <ol start="3" type="A"> <li>Troisième point (affiché C)</li> <li>Quatrième point (affiché D)</li> </ol> <!-- Liste de définitions --> <dl> <dt>HTML</dt> <dd>Langage de balisage pour structurer le web</dd> <dt>CSS</dt> <dd>Langage de style pour mettre en forme le web</dd> </dl>
📌 L'attribut type de <ol> accepte : 1 (défaut), A, a, I (chiffres romains majuscules), i. L'attribut start définit le numéro de départ.
🔗 Liens, médias & intégrations
7
Les liens — <a> et ses attributs

La balise <a> (anchor) est l'une des plus puissantes du HTML. Elle ne sert pas qu'à relier des pages.

href
URL de destination. Peut être absolue (https://...), relative (./page.html), ou une ancre (#section).
target
_blank ouvre dans un nouvel onglet. _self (défaut), _parent, _top.
rel
noopener noreferrer pour les liens externes (sécurité). nofollow pour les moteurs de recherche.
download
Force le téléchargement du fichier lié au lieu de l'ouvrir.
href="mailto:"
Ouvre le client mail avec l'adresse pré-remplie.
href="tel:"
Déclenche un appel sur mobile.
<!-- Lien externe sécurisé --> <a href="https://w3schools.com" target="_blank" rel="noopener noreferrer"> W3Schools </a> <!-- Ancre interne — lien vers une section de la même page --> <a href="#contact">Aller à la section contact</a> <section id="contact">...</section> <!-- Téléchargement de fichier --> <a href="rapport.pdf" download="mon-rapport">Télécharger le PDF</a> <!-- Lien email et téléphone --> <a href="mailto:contact@site.fr">Envoyer un email</a> <a href="tel:+33612345678">Appeler</a>
8
Images, audio, vidéo

HTML5 intègre nativement l'audio et la vidéo sans plugin. La balise <picture> permet de servir des images différentes selon l'écran.

<!-- Image simple avec attributs essentiels --> <img src="photo.jpg" alt="Description pour les malvoyants et Google" width="800" height="600" loading="lazy" > <!-- Image responsive avec <picture> --> <picture> <source media="(max-width: 600px)" srcset="small.jpg"> <source media="(max-width: 1200px)" srcset="medium.jpg"> <img src="large.jpg" alt="Photo adaptative"> </picture> <!-- Vidéo HTML5 native --> <video controls autoplay muted loop poster="apercu.jpg"> <source src="video.mp4" type="video/mp4"> <source src="video.webm" type="video/webm"> Ton navigateur ne supporte pas la vidéo HTML5. </video> <!-- Audio HTML5 native --> <audio controls preload="metadata"> <source src="son.mp3" type="audio/mpeg"> </audio>
⚠️ L'attribut alt sur <img> est obligatoire. Vide (alt="") pour les images décoratives, descriptif pour les images de contenu. C'est essentiel pour l'accessibilité et le référencement.
📋 Formulaires avancés
9
Les formulaires — <form> et ses éléments

Les formulaires sont l'un des sujets les plus riches du HTML. HTML5 a introduit de nombreux types d'<input> qui apportent des comportements natifs sans JavaScript.

<!-- Structure d'un formulaire complet --> <form action="/envoyer" method="POST" novalidate> <!-- Label TOUJOURS associé à son input via "for" / "id" --> <label for="nom">Nom complet</label> <input type="text" id="nom" name="nom" placeholder="Jean Dupont" required autocomplete="name"> <label for="email">Email</label> <input type="email" id="email" name="email" required> <label for="age">Âge</label> <input type="number" id="age" name="age" min="0" max="120" step="1"> <label for="couleur">Couleur préférée</label> <input type="color" id="couleur" name="couleur" value="#5DCAA5"> <label for="niveau">Niveau (0-100)</label> <input type="range" id="niveau" min="0" max="100" value="50"> <input type="submit" value="Envoyer"> </form>

Tous les types d'<input> HTML5 :

text · password · email · url
number · range · tel · search
date · time · datetime-local
month · week · color
checkbox · radio · file
submit · reset · button
hidden · image
10
Éléments de formulaire avancés
<!-- Liste de suggestions avec <datalist> --> <input type="text" list="villes" placeholder="Choisir une ville"> <datalist id="villes"> <option value="Paris"> <option value="Lyon"> <option value="Marseille"> </datalist> <!-- Groupement de champs avec <fieldset> --> <fieldset> <legend>Informations personnelles</legend> <input type="text" name="prenom" placeholder="Prénom"> <input type="text" name="nom" placeholder="Nom"> </fieldset> <!-- Barre de progression avec <progress> et <meter> --> <progress value="70" max="100">70%</progress> <meter value="6" min="0" max="10" low="3" high="7" optimum="9">6/10</meter>
🎯 <progress> représente une tâche en cours (chargement). <meter> représente une mesure dans une plage connue (score, niveau de batterie). Les deux s'affichent nativement sans CSS.
🖱️ Balises interactives HTML5
11
Details, summary, dialog — interactivité sans JS

HTML5 offre des éléments interactifs natifs — sans une seule ligne de JavaScript. Ils sont sous-utilisés mais très puissants.

BaliseCe qu'elle fait
<details>
Accordion natif. Contenu masqué/révélé au clic. L'attribut open l'affiche par défaut.
<summary>
Titre cliquable d'un <details>. Remplace le triangle par défaut.
<dialog>
Fenêtre modale native. L'attribut open l'affiche. JS : .showModal() / .close().
<menu>
Groupe de boutons ou d'actions. Sémantiquement différent de <ul>.
<!-- Accordion 100% HTML —aucun JS --> <details> <summary>❓ Qu'est-ce que le HTML ?</summary> <p>Le HTML est un langage de balisage pour le web.</p> </details> <details open> <!-- ouvert par défaut --> <summary>💡 Astuce importante</summary> <p>Cette section est visible dès le chargement.</p> </details> <!-- Modale native HTML5 --> <dialog id="maModale"> <h2>Titre de la modale</h2> <p>Contenu...</p> <button onclick="document.getElementById('maModale').close()">Fermer</button> </dialog> <button onclick="document.getElementById('maModale').showModal()">Ouvrir</button>
12
Les tableaux — <table> et sa structure complète

Les tableaux HTML servent à afficher des données tabulaires — et uniquement ça. Jamais pour la mise en page (c'est le rôle du CSS).

<table> <caption>Résultats du trimestre</caption> <!-- titre du tableau --> <thead> <!-- en-têtes de colonnes --> <tr> <th scope="col">Matière</th> <th scope="col">Note</th> </tr> </thead> <tbody> <!-- données --> <tr> <td>HTML</td> <td>18/20</td> </tr> <tr> <td colspan="2">Mention très bien</td> <!-- fusionne 2 colonnes --> </tr> </tbody> <tfoot> <!-- pied de tableau --> <tr> <td>Moyenne</td> <td>17/20</td> </tr> </tfoot> </table>
📌 colspan="2" fusionne 2 colonnes horizontalement. rowspan="2" fusionne 2 lignes verticalement. L'attribut scope sur <th> améliore l'accessibilité.
🔧 Attributs globaux essentiels
13
Attributs disponibles sur toutes les balises

Ces attributs s'appliquent à n'importe quelle balise HTML. Ils sont fondamentaux pour le CSS, le JavaScript et l'accessibilité.

id
Identifiant unique dans la page. Utilisé par le CSS (#mon-id), JavaScript (getElementById), et les ancres de lien (#mon-id). Un seul par page.
class
Classe(s) CSS. Plusieurs classes séparées par des espaces : class="card active highlight". Réutilisable sur plusieurs éléments.
style
CSS inline directement sur l'élément. À éviter en général — préférer les classes CSS pour maintenir la séparation contenu/style.
data-*
Attributs de données personnalisés. Stockent des infos pour JavaScript : data-user-id="42", accessible via element.dataset.userId.
hidden
Masque l'élément (équivalent CSS display:none). Attribut booléen — sa simple présence suffit.
title
Infobulle au survol. Texte affiché dans une tooltip par le navigateur.
tabindex
Contrôle la navigation au clavier (touche Tab). 0 = dans l'ordre naturel, -1 = accessible par JS mais pas par Tab, 1+ = ordre personnalisé.
contenteditable
"true" rend n'importe quel élément éditable directement dans la page, comme un champ texte.
draggable
"true" rend l'élément déplaçable par glisser-déposer. Utilisé avec les événements JS dragstart, drop.
lang
Langue du contenu. Sur <html lang="fr"> pour la page entière, ou sur un élément pour un passage en langue différente.
spellcheck
"true" ou "false" — active ou désactive le correcteur orthographique natif sur les champs editables.
<!-- Exemples d'attributs data-* --> <div id="carte-1" class="carte active" data-product-id="42" data-price="29.99" data-category="electronique" > Produit </div> // Accès en JavaScript : // element.dataset.productId → "42" // element.dataset.price → "29.99"
14
Les balises <meta> — SEO & réseaux sociaux

Les balises <meta> dans le <head> transmettent des informations aux moteurs de recherche, aux réseaux sociaux et aux navigateurs.

<head> <!-- Essentiels --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1"> <meta name="description" content="Description de 150-160 caractères pour Google"> <meta name="author" content="Ton nom"> <meta name="robots" content="index, follow"> <!-- Open Graph (Facebook, LinkedIn, Discord...) --> <meta property="og:title" content="Titre affiché sur les réseaux"> <meta property="og:description" content="Description courte"> <meta property="og:image" content="https://monsite.fr/image.jpg"> <meta property="og:url" content="https://monsite.fr"> <!-- Twitter Card --> <meta name="twitter:card" content="summary_large_image"> <meta name="twitter:title" content="Titre pour Twitter"> </head>
🔑 La balise description n'impacte pas directement le classement Google — mais elle influe sur le taux de clic dans les résultats de recherche. Soigne-la.

Où découvrir et tester d'autres balises

Il existe plus de 100 balises HTML. Tu n'as pas besoin de toutes les connaître — tu as besoin de savoir où les trouver et comment les tester. Voici les meilleures ressources, toutes gratuites.

🏫
W3Schools

🦊

La documentation officielle la plus complète et fiable. En français.
developer.mozilla.org
📖
WHATWG Spec
La spécification officielle du HTML vivant. Pour les curieux avancés.
html.spec.whatwg.org
🖊️
CodePen
Éditeur en ligne pour tester HTML/CSS/JS et partager des extraits de code.
codepen.io
✅
W3C Validator
Valide ton HTML. Détecte les balises mal fermées et les erreurs de syntaxe.
validator.w3.org
🌐
Can I Use
Vérifie la compatibilité de n'importe quelle balise ou fonctionnalité HTML avec les navigateurs.
caniuse.com
Comment utiliser W3Schools efficacement
1. Cherche la balise qui t'intéresse (ex: "html details tag")
2. Lis la description et les attributs disponibles
3. Clique sur "Try It Yourself" pour voir et modifier l'exemple en direct
4. Expérimente en changeant les valeurs des attributs
5. Copie ce que tu as compris dans ton propre projet
⚠️ W3Schools est excellent pour découvrir et tester. Pour les informations les plus précises et à jour, confirme toujours sur MDN. Les deux se complètent.
