C'est quoi un terminal ?

Tu connais l'explorateur de fichiers : la fenêtre où tu vois tes dossiers et tu cliques dessus pour les ouvrir. Le terminal, c'est exactement la même chose, mais au lieu de cliquer, tu tapes des commandes au clavier !

🗺️
Imagine que ton PC est une maison géante. L'explorateur de fichiers, c'est comme te balader dans la maison en regardant les pièces. Le terminal, c'est comme avoir un assistant à qui tu peux dire "va dans la cuisine", "montre-moi ce qu'il y a dans le frigo", "crée un nouveau tiroir". C'est plus rapide quand on sait s'en servir !
💡Au début ça fait peur, mais c'est comme apprendre des mots magiques. Une fois qu'on connaît 5 commandes, on peut déjà faire plein de choses !
T
Les différents types de terminaux

Il existe plusieurs "modèles" de terminal selon ton système d'exploitation. Ils font la même chose, mais les commandes ont parfois des petites différences !

🪟 Windows
PowerShell
Le plus moderne & recommandé.
Préinstallé sur Windows 10/11.
Tape powershell dans la barre de recherche.
🪟 Windows (ancien)
CMD (Invite de commandes)
L'ancien terminal Windows.
Commandes un peu différentes.
Tape cmd dans la recherche.
🍎 macOS
Terminal
Préinstallé sur Mac.
Recherche "Terminal" dans Spotlight.
Ou : Applications → Utilitaires.
🐧 Linux
Bash / Zsh
Préinstallé, souvent appelé "Konsole" ou "Terminal".
Raccourci : Ctrl+Alt+T.
🌟Bonne nouvelle : si tu utilises VS Code, il y a un terminal intégré ! Clique sur "Terminal" dans le menu en haut, puis "Nouveau terminal". C'est souvent le plus pratique pour coder.

Comment ouvrir son terminal ?
Sur Windows
1
Appuie sur la touche Windows (⊞) et tape powershell
2
Clique sur "Windows PowerShell" dans les résultats
3
Une fenêtre noire (ou bleue) avec du texte s'ouvre — c'est bon !
Sur Mac
1
Appuie sur Cmd + Espace pour ouvrir Spotlight
2
Tape Terminal et appuie sur Entrée
3
Une fenêtre avec un fond blanc ou noir apparaît — c'est bon !
Dans VS Code (tous les systèmes)
1
Ouvre VS Code
2
Dans le menu en haut : Terminal → Nouveau terminal
3
Le terminal apparaît en bas de la fenêtre
⚠️Quand le terminal est ouvert, tu vois une invite de commandes : un texte qui se termine par > ou $. C'est là que tu tapes tes commandes !
📍
Se repérer : où suis-je ?

Quand tu ouvres le terminal, tu te trouves quelque part dans ton PC (souvent dans ton dossier personnel). La première chose à apprendre, c'est savoir où tu es !

Afficher le dossier actuel
$ pwd # Mac/Linux : "Print Working Directory" /Users/alice/Documents > cd # Windows PowerShell : tape "cd" sans argument C:\Users\alice\Documents
💡pwd te dit exactement où tu te trouves. C'est comme demander "Dans quelle pièce de la maison je suis ?"
👀
Voir le contenu d'un dossier

Pour voir ce qu'il y a dans le dossier où tu te trouves, tu utilises une commande qui liste tous les fichiers et sous-dossiers.

Lister les fichiers
# Sur Mac/Linux : $ ls Documents Photos Projets index.html style.css # Sur Windows PowerShell : > ls # fonctionne aussi dans PowerShell ! Répertoire : C:\Users\alice Mode Nom ---- --- d---- Documents d---- Photos -a--- index.html # Voir PLUS de détails (taille, date) : $ ls -l # Mac/Linux > dir # Windows CMD (commande alternative)
💡ls vient de "list". C'est comme ouvrir un dossier dans l'explorateur et voir tous les fichiers à l'intérieur !
📂
Se déplacer entre les dossiers

La commande cd (Change Directory) te permet de changer de dossier. C'est la commande la plus utilisée dans le terminal !

# Entrer dans un sous-dossier $ cd Documents $ pwd /Users/alice/Documents # Entrer dans un dossier encore plus profond $ cd Projets/mon-site # Tu es maintenant dans /Users/alice/Documents/Projets/mon-site # Remonter d'UN dossier (aller au parent) $ cd .. # Tu es maintenant dans /Users/alice/Documents/Projets # Remonter de DEUX dossiers $ cd ../.. # Tu es maintenant dans /Users/alice/Documents # Aller directement à la racine de ton dossier perso $ cd ~ # ~ = dossier personnel (Mac/Linux/PowerShell)
🌟Astuce ! Tape les premières lettres d'un dossier puis appuie sur Tab ↹ : le terminal complète automatiquement le nom ! Par exemple tape cd Doc puis Tab → ça devient cd Documents.
💡Le .. signifie "le dossier parent" — exactement comme dans les chemins HTML ! Si tu as compris le cours sur l'architecture, tu connais déjà ce concept.

Pourquoi le terminal est utile pour Git ?

Tu vas bientôt apprendre Git, un outil pour sauvegarder et partager ton code. Git s'utilise principalement dans le terminal ! Voici pourquoi maîtriser le terminal d'abord est important :

1
Vérifier si Git est installé : tu taperas une commande dans le terminal pour le savoir
2
Se placer dans le bon dossier : avant de faire quoi que ce soit avec Git, tu dois naviguer dans ton projet avec cd
3
Toutes les commandes Git se tapent dans le terminal : git add, git commit, git push…
# Exemple : je veux travailler sur mon projet "mon-site" $ cd Documents/Projets/mon-site $ ls index.html Assets/ pages/ # Parfait, je suis dans le bon dossier. Maintenant je peux utiliser Git ! $ git status
🚀Le terminal et Git vont ensemble : plus tu t'entraînes à naviguer dans le terminal, plus Git sera simple à utiliser ensuite !
!
Les erreurs courantes

Tout le monde fait ces erreurs au début — c'est normal !

❌ Le dossier n'existe pas
cd Documants ← faute de frappe !
bash: cd: Documants: No such file or directory
Vérifie l'orthographe avec ls d'abord, ou utilise la touche Tab pour auto-compléter.
❌ Majuscules/minuscules incorrectes
cd documents ← si le dossier s'appelle "Documents"
Sur Mac et Linux, Documents ≠ documents. Sur Windows, c'est plus tolérant, mais prends l'habitude d'être précis.
❌ Dossier avec des espaces
cd Mon Projet ← le terminal croit que ce sont deux dossiers !
cd "Mon Projet" ← utilise des guillemets
C'est pour ça qu'on évite les espaces dans les noms de dossiers et fichiers de projets !
