Mets en pratique la documentation de tes versions !

3 exercices Interactif
1
Exercice 1 — Comprendre les numéros de version

Pour chaque question, choisis la bonne réponse sur le versioning sémantique.

Tu ajoutes une toute petite modification à ton projet (correction d'une faute de frappe). Quel numéro de version s'incrémente ?
Tu ajoutes une nouvelle page entière à ton site. Quel numéro de version s'incrémente ?
Après v0.00.9, quelle est la version suivante si tu ajoutes une nouvelle fonctionnalité ?
Ton projet est terminé et tu es prêt à le présenter au public. Quel est le bon numéro de version majeure ?
Dans un patch-note, quel symbole utilise-t-on pour indiquer la correction d'un bug ?
2
Exercice 2 — Remets les versions dans l'ordre chronologique

Ces versions de patch-note sont dans le désordre ! Glisse-les pour les remettre de la plus ancienne (en haut) à la plus récente (en bas).

💡Pour ordonner les versions : compare d'abord le premier chiffre, puis le deuxième, puis le troisième. v0.02.0 vient avant v1.00.0 car 0 < 1.
3
Exercice 3 — Associe chaque changement au bon symbole

Voici des changements qui auraient pu être dans un patch-note. Pour chaque ligne, choisis si c'est un ajout +, une correction *, ou une modification ~.
